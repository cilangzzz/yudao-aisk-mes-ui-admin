# 班次管理模块 API 接口文档

## 1. 模块概述

班次管理模块用于管理企业的工作班次信息，定义一天中的工作时段。班次是时间维度的概念，与班组配合使用实现生产人员的排班管理。该模块支持多种班制配置（如单班制、两班制、三班制），是生产排程、考勤管理、产能计算的基础数据支撑。

**主要功能**：
- 班次信息的创建、更新、删除和查询
- 班次分页列表和全量列表查询
- 班次数据 Excel 导出

**业务场景**：
- 定义工作班次时间段
- 支持生产排程的时间分配
- 支持人员考勤管理
- 计算班次产能

## 3. 接口详细说明

### 3.1 创建班次

#### 基本信息
- **路径**: `/mes/shift/create`
- **方法**: POST
- **描述**: 创建新的班次信息
- **权限**: `mes:shift:create`

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 | 示例值 |
|--------|------|------|------|--------|
| shiftCode | String | 是 | 班次编码 | SHIFT001 |
| shiftName | String | 是 | 班次名称 | 白班 |
| startTime | String | 是 | 开始时间（HH:mm格式） | 08:00 |
| endTime | String | 是 | 结束时间（HH:mm格式） | 16:00 |
| status | Integer | 否 | 状态（0=启用，1=禁用） | 0 |

#### 请求示例

```json
{
  "shiftCode": "SHIFT001",
  "shiftName": "白班",
  "startTime": "08:00",
  "endTime": "16:00",
  "status": 0
}
```

#### 响应参数

| 参数名 | 类型 | 说明 |
|--------|------|------|
| code | Integer | 状态码，0表示成功 |
| data | Long | 班次ID |
| msg | String | 提示信息 |

#### 响应示例

```json
{
  "code": 0,
  "data": 1,
  "msg": "操作成功"
}
```

## 6. 时间格式说明

### 时间字段格式

| 字段 | 格式 | 示例 | 说明 |
|------|------|------|------|
| startTime | HH:mm | 08:00 | 班次开始时间 |
| endTime | HH:mm | 16:00 | 班次结束时间 |

### 跨天班次处理

当结束时间小于开始时间时，表示跨天班次。例如：

| 班次名称 | 开始时间 | 结束时间 | 实际时段 |
|----------|----------|----------|----------|
| 夜班 | 22:00 | 06:00 | 当日22:00 - 次日06:00 |

## 7. 常见班制配置

### 单班制

适用于白天生产的工厂：

| 班次编码 | 班次名称 | 开始时间 | 结束时间 | 工作时长 |
|----------|----------|----------|----------|----------|
| SHIFT001 | 白班 | 08:00 | 17:00 | 8小时（含1小时休息） |

### 两班制

适用于日班+夜班的生产模式：

| 班次编码 | 班次名称 | 开始时间 | 结束时间 | 工作时长 |
|----------|----------|----------|----------|----------|
| SHIFT001 | 白班 | 08:00 | 20:00 | 12小时 |
| SHIFT002 | 夜班 | 20:00 | 08:00 | 12小时 |

### 三班制

适用于24小时连续生产：

| 班次编码 | 班次名称 | 开始时间 | 结束时间 | 工作时长 |
|----------|----------|----------|----------|----------|
| SHIFT001 | 白班 | 08:00 | 16:00 | 8小时 |
| SHIFT002 | 中班 | 16:00 | 00:00 | 8小时 |
| SHIFT003 | 夜班 | 00:00 | 08:00 | 8小时 |

### 四班三倒

适用于需要休息轮换的生产模式：

| 班次编码 | 班次名称 | 开始时间 | 结束时间 | 工作时长 |
|----------|----------|----------|----------|----------|
| SHIFT001 | 白班 | 08:00 | 16:00 | 8小时 |
| SHIFT002 | 中班 | 16:00 | 00:00 | 8小时 |
| SHIFT003 | 夜班 | 00:00 | 08:00 | 8小时 |

> 说明：四班三倒模式下，设置4个班组轮流上3个班次，保证每班有班组在休息。

## 8. 业务规则

### 删除规则
- 存在关联的排班记录时，不允许删除班次
- 存在关联的考勤记录时，不允许删除班次

### 编码规则
- 班次编码唯一，不可重复
- 建议编码格式：SHIFT + 序号，如 SHIFT001、SHIFT002

### 时间校验规则
- 开始时间和结束时间不能相同
- 时间格式必须为 HH:mm

### 班次时长计算

| 场景 | 计算方式 | 示例 |
|------|----------|------|
| 同一天 | 结束时间 - 开始时间 | 08:00-16:00 = 8小时 |
| 跨天 | (24:00 - 开始时间) + 结束时间 | 22:00-06:00 = 8小时 |

## 10. 班次与班组的配合使用

### 概念区分

| 概念 | 定义 | 示例 |
|------|------|------|
| 班次 | 时间概念，定义工作时段 | 白班（08:00-16:00） |
| 班组 | 人员概念，定义作业团队 | 甲班（张三、李四等） |

### 排班示例

假设采用三班制，设置甲、乙、丙三个班组：

| 日期 | 白班（08:00-16:00） | 中班（16:00-00:00） | 夜班（00:00-08:00） |
|------|---------------------|---------------------|---------------------|
| 周一 | 甲班 | 乙班 | 丙班 |
| 周二 | 丙班 | 甲班 | 乙班 |
| 周三 | 乙班 | 丙班 | 甲班 |
| 周四 | 甲班 | 乙班 | 丙班 |
| ... | ... | ... | ... |

> 说明：通过班组和班次的组合，实现人员的轮班安排，保证生产的连续性和员工的休息需求。
